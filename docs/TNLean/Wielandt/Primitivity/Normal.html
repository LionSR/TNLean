    Primitivity consequences and normality restatements #

    This file records the low-level consequences of HasPrimitiveFixedPoint used in the Wielandt development and recalls the normality side of the chain from WielandtBound.lean.

    It does not prove HasPrimitiveFixedPoint → IsNormal. The currently formalized bridge with extra ρ.PosDef and aperiodicity hypotheses is assembled in QuantumWielandt.lean; the unconditional implication remains future work.

    Main results #

    From primitivity #

    • transferMap_pow_fixed: if E(ρ) = ρ, then E^n(ρ) = ρ
    • transferMap_pow_apply_eq_sum: E^n(X) = Σ_σ (evalWord A σ) X (evalWord A σ)†
    • exists_nonzero_evalWord_of_isPrimitiveMPS: for every n, some length-n word product is nonzero
    • exists_nonzero_evalWord_of_hasPrimitiveFixedPoint: existential wrapper
    • transferMap_pow_ne_zero_of_hasPrimitiveFixedPoint: every transfer-map iterate is nonzero

    From normality #

    • wielandt_full_analysis: packages the four standard Wielandt-chain outputs

    References #

    • Sanz, Pérez-García, Wolf, Cirac, A quantum version of Wielandt's inequality, arXiv:0909.5347, Proposition 3

    Part 1: Transfer map fixed-point iteration #

    source
    theorem MPSTensor.transferMap_pow_fixed {d D : ℕ} {A : MPSTensor d D} {ρ : Matrix (Fin D) (Fin D) ℂ} (hfix : A.transferMap ρ = ρ) (n : ℕ) :
    (A.transferMap ^ n) ρ = ρ

    If transferMap A ρ = ρ, then (transferMap A)^n ρ = ρ for all n.

    This is the iteration of the fixed-point equation.

    theorem MPSTensor.transferMap_pow_apply_eq_sum {d D : ℕ} (A : MPSTensor d D) (n : ℕ) (X : Matrix (Fin D) (Fin D) ℂ) :
    (A.transferMap ^ n) X = ∑ σ : Fin n → Fin d, A.evalWord (List.ofFn σ) * X * (A.evalWord (List.ofFn σ)).conjTranspose

    The iterated transfer map expands as a sum over word products. Re-states transferMap_pow_apply' in a form convenient for our arguments.

    Part 2: Nonzero word products from primitivity #

    source
    theorem MPSTensor.exists_nonzero_evalWord_of_isPrimitiveMPS {d D : ℕ} [NeZero D] {A : MPSTensor d D} {ρ : Matrix (Fin D) (Fin D) ℂ} (hP : A.IsPrimitiveMPS ρ) (n : ℕ) :
    ∃ (σ : Fin n → Fin d), A.evalWord (List.ofFn σ) ≠ 0

    Every word length has a nonzero word product under primitivity.

    Given IsPrimitiveMPS A ρ (transfer map has a spectral gap with PSD fixed point ρ), for every n : ℕ there exists a word σ : Fin n → Fin d such that evalWord A (List.ofFn σ) ≠ 0.

    Proof: Since E^n(ρ) = ρ ≠ 0 and E^n(ρ) = Σ_σ (evalWord A σ) ρ (evalWord A σ)†, the sum is nonzero, so at least one summand is nonzero, hence some evalWord A σ ≠ 0.

    Paper: implicit in arXiv:0909.5347, Section III.

    Part 3: Recall the Wielandt chain from normality #

    source
    theorem MPSTensor.wielandt_full_analysis {d D : ℕ} [NeZero D] (A : MPSTensor d D) (hN : A.IsNormal) :
    A.cumulativeSpan (D ^ 2) = ⊤ ∧ (∃ (w₀ : List (Fin d)), w₀.length ≤ D ^ 2 ∧ (A.evalWord w₀).trace ≠ 0) ∧ (∃ (w₀ : List (Fin d)) (μ : ℂ) (φ : Fin D → ℂ), w₀.length ≤ D ^ 2 ∧ μ ≠ 0 ∧ φ ≠ 0 ∧ (A.evalWord w₀).mulVec φ = μ • φ) ∧ ∀ (φ : Fin D → ℂ), φ ≠ 0 → A.cumulativeVectorSpan φ (D ^ 2) = ⊤

    The complete Wielandt analysis from normality.

    Given IsNormal A, the full Wielandt chain provides:

    1. Cumulative span T_{D²} = ⊤ (all matrices reachable)
    2. A word w₀ with |w₀| ≤ D² and tr(evalWord A w₀) ≠ 0
    3. A nonzero eigenvalue μ and eigenvector φ for evalWord A w₀
    4. Vector spanning: for any nonzero φ, word products applied to φ span ℂ^D

    This records wielandt_chain from WielandtBound.lean under a cleaner name.

    Part 4: Status of the primitive → normal bridge #

    The unconditional implication HasPrimitiveFixedPoint → IsNormal is still open in this file. The currently formalized route in QuantumWielandt.lean proves IsNormal from IsPrimitiveMPS A ρ under the extra hypotheses that ρ is positive definite and 1 ∈ wordSpan A 1.

    What remains here is the gap from bare spectral-gap primitivity to eventual full matrix spanning. Conceptually this should follow from irreducibility plus a Burnside- or peripheral-spectrum argument, but that infrastructure is kept out of this lightweight wrapper file.
