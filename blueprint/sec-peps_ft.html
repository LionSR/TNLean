18 Fundamental Theorem for Injective PEPS

The following definitions and results extend the Fundamental Theorem to injective PEPS on simple graphs, following Theorem 2 of  [ MGRPG+18 ] .

Definition 18.1 Graph edge for PEPS
✓
#

Lean declarations

  • TNLean.PEPS.Edge

For a finite simple graph \(G\), an edge is an ordered pair \((u,v)\) with \(u {\lt} v\) and \(u\) adjacent to \(v\). The ordering is only used to avoid double counting.

Definition 18.2 Incident edges at a vertex
✓
#

Uses

  • Definition 18.1

Lean declarations

  • TNLean.PEPS.IncidentEdge

For a vertex \(v\), an incident edge is an edge of \(G\) having \(v\) as one of its endpoints.

Definition 18.3 PEPS tensor on a graph
✓
#

Uses

  • Definition 18.1
  • Definition 18.2

Lean declarations

  • TNLean.PEPS.Tensor

A PEPS tensor on \(G\) with physical dimension \(d\) consists of a bond dimension \(D_e\) for every edge \(e\), together with a local tensor at each vertex \(v\) whose virtual indices are indexed by the edges incident to \(v\) and whose physical index lies in \(\{ 0,\ldots ,d-1\} \).

Definition 18.4 Virtual configuration
✓
#

Uses

  • Definition 18.3
  • Definition 18.1

Lean declarations

  • TNLean.PEPS.VirtualConfig

A virtual configuration assigns to every edge \(e\) one basis index in the corresponding bond space of dimension \(D_e\).

Definition 18.5 PEPS state coefficient
✓
#

Uses

  • Definition 18.3
  • Definition 18.4

Lean declarations

  • TNLean.PEPS.stateCoeff

For a physical configuration \(\sigma \) on the vertices, the PEPS state coefficient is obtained by summing over all virtual configurations and, for each such configuration, multiplying the local tensor entries over all vertices.

Definition 18.6 Equality of PEPS states
✓
#

Uses

  • Definition 18.5

Lean declarations

  • TNLean.PEPS.SameState

Two PEPS tensors represent the same state if all their state coefficients agree for every physical configuration.

Definition 18.13 Local tensor evaluation
✓
#

Lean declarations

  • TNLean.PEPS.localTensorEval

The local tensor evaluated at vertex \(v\) with virtual-index weighting \(f\), computing \(\sum _\eta (\prod _{ie} f(ie)(\eta (ie))) \cdot A_v(\eta , \sigma )\). This is multilinear in the components of \(f\), not linear in the full tuple.

Definition 18.14 Vertex injectivity
✓
#

Lean declarations

  • TNLean.PEPS.IsVertexInjective

A PEPS tensor \(A\) is vertex-injective if, at every vertex \(v\), the family of physical vectors

\[ \bigl(A_v(\eta , \cdot ) \in \mathbb {C}^d\bigr)_{\eta } \]

indexed by virtual configurations \(\eta \) on the edges incident to \(v\) is linearly independent in \(\mathbb {C}^d\). Equivalently, extending \(A_v\) by linearity to a map from the free vector space on virtual configurations into the physical space \(\mathbb {C}^d\) gives a linear map with trivial kernel. This matches the paper’s formulation that "all tensors interpreted as maps from the virtual space to the physical one are injective" ( [ MGRPG+18 ] , §3).

Definition 18.15 Local tensor map
✓
#

Uses

  • Definition 18.3
  • Definition 18.2

Lean declarations

  • TNLean.PEPS.localTensorMap

For a fixed vertex \(v\), the family of local physical vectors \(\eta \mapsto A_v(\eta , \cdot )\) extends by linearity to a map from the coefficient space on local virtual configurations at \(v\) into \(\mathbb {C}^d\).

Definition 18.16 Local left inverse
✓
#

Uses

  • Definition 18.15
  • Definition 18.14

Lean declarations

  • TNLean.PEPS.localLeftInverse

If \(A\) is vertex-injective, one may choose a linear map \(\Psi _v\) from \(\mathbb {C}^d\) to the coefficient space on local virtual configurations at \(v\) such that \(\Psi _v \circ \Phi _v = \mathbb {1}\).

✓

Uses

  • TNLean.PEPS.physRealizeLocalOp

For an endomorphism \(T\) of the coefficient space on local virtual configurations at \(v\), define the local physical operator

\[ O_T = \Phi _v \circ T \circ \Psi _v . \]
Theorem 18.18 Local virtual operations on the image
✓
#

Uses

  • Definition 18.17

Lean declarations

  • TNLean.PEPS.physRealizeLocalOp_spec

For every coefficient function \(c\) on the local virtual configurations at \(v\),

\[ O_T\bigl(\Phi _v(c)\bigr) = \Phi _v(Tc). \]

Definition 18.20 Middle region of an edge
✓
#

Uses

  • Definition 18.1

Lean declarations

  • TNLean.PEPS.edgeMiddleVertices

For an edge \(e = (u,v)\), the middle region is the complement \(V \setminus \{ u,v\} \).

Theorem 18.21 Edge-centered vertex partition
✓
#

Uses

  • Definition 18.20

Lean declarations

  • TNLean.PEPS.edgeVertices_union

For an edge \(e = (u,v)\), the three regions \(\{ u\} \), \(V \setminus \{ u,v\} \), and \(\{ v\} \) cover the whole vertex set.

Proof ▶

Every vertex is either \(u\), \(v\), or different from both.

Theorem 18.22 Product splitting at an edge
✓
#

Uses

  • Definition 18.20
  • Theorem 18.21

Lean declarations

  • TNLean.PEPS.prod_univ_splitAtEdge

For any edge \(e = (u,v)\) and any function \(f\) on the vertex set with values in a commutative monoid \(M\),

\[ \prod _{x \in V} f(x) = f(u) \Bigl(\prod _{x \in V \setminus \{ u,v\} } f(x)\Bigr) f(v). \]
Proof ▶

Isolate the two distinguished endpoints in the product over \(V\) and group the remaining factors into the middle region.

Theorem 18.23 PEPS coefficient splitting at an edge
✓
#

Uses

  • Definition 18.5
  • Definition 18.20
  • Theorem 18.22

Lean declarations

  • TNLean.PEPS.stateCoeff_splitAtEdge

For any edge \(e = (u,v)\), each summand in the PEPS coefficient factors into the tensor contribution at \(u\), the product over the middle region \(V \setminus \{ u,v\} \), and the tensor contribution at \(v\).

Proof ▶

Expand the state coefficient and apply the product splitting at the chosen edge to the per-vertex product inside each summand.

Definition 18.24 Sharpened local gauge lift datum
✓
#

Uses

  • Definition 18.15
  • Definition 18.16

Lean declarations

  • TNLean.PEPS.HasLocalGaugeLift

For fixed \(A\), \(B\), a bond-dimension equality, and a vertex \(v\), this records the two local conclusions still needed from the blocked-middle reduction in  [ MGRPG+18 ] , §3: (i) each local component of \(B\) lies in the image of the local tensor map of \(A\), and (ii) the canonical pullback through the chosen left inverse of \(A\) factorizes as one invertible matrix on each incident edge.

Definition 18.25 Blocked-middle local gauge output
✓
#

Uses

  • Definition 18.24

  • TNLean.PEPS.BlockedMiddleGaugeHyp

This states the explicit local gauge formula that the edge-centered blocked-middle reduction is expected to produce: a family of invertible matrices on the incident edges of \(v\) whose product kernel reconstructs each local component of \(B\) from those of \(A\).

Theorem 18.26 Blocked-middle output implies sharpened lift
✓
#

Uses

  • Definition 18.25
  • Definition 18.24

  • TNLean.PEPS.hasLocalGaugeLift_of_blockedMiddleGaugeHyp

Any explicit local gauge formula from the blocked-middle reduction yields the sharpened local lift. The remaining open step is to derive this blocked-middle formula from equality of PEPS states.

Proof ▶

The explicit local gauge formula already places each local component of \(B\) in the image of the local tensor map of \(A\). Applying the chosen left inverse of \(A\) identifies the canonical candidate with the same factorized coefficient kernel, so both parts of the sharpened lift follow.

#

Uses

  • Definition 18.10
  • Definition 18.14

If two PEPS tensors on a simple graph are both vertex-injective and generate the same state, then they are gauge-equivalent (Theorem 2 of  [ MGRPG+18 ] ).

Definition 18.31 Gauge equivalence modulo balanced edge scalars
✓
#

Uses

  • Definition 18.7

Lean declarations

  • TNLean.PEPS.GaugeEquivModEdgeScalars

Two gauge families are equivalent modulo balanced edge scalars if there is a nonzero scalar \(c_e\) on each edge such that the corresponding endpoint actions rescale the oriented edge gauges, and the oriented product of those endpoint scalars is \(1\) at every vertex.

Theorem 18.32 Balanced edge scalars preserve the local gauge action
✓
#

Uses

  • Definition 18.31
  • Definition 18.8
  • Definition 18.31
  • Definition 18.8

Lean declarations

  • TNLean.PEPS.GaugeEquivModEdgeScalars.gaugeVertex_eq

If two gauge families differ by a balanced edge-scalar reweighting, then they produce the same gauged tensor at every vertex.

Proof ▶

In the defining sum for the gauged tensor, each incident edge contributes its endpoint scalar factor in addition to the original gauge entry. The product of these endpoint scalars is \(1\) by the balancing condition, so every summand is unchanged.

Theorem 18.33 Gauge uniqueness modulo balanced edge scalars
#

Uses

  • Definition 18.31
  • Definition 18.14

Lean declarations

  • TNLean.PEPS.gauge_unique_mod_edge_scalars

If two gauge families relate the same injective PEPS tensor to the same target tensor, then they define the same class modulo balanced edge scalars. This replaces the earlier global-scalar claim, which is false on a triangle with bond dimension \(1\).

Remark 18.34
#

The local left inverse \(\Psi _v\), the realization map \(T \mapsto O_T\), the split at one incident edge, and the partition \(V = \{ u\} \sqcup (V \setminus \{ u,v\} ) \sqcup \{ v\} \) isolate the local data needed for the edge-centered reduction of   [ MGRPG+18 ] , §3. The remaining three PEPS theorems still require the blocked tensor on the middle region and the comparison with the corresponding three-site injective chain. The uniqueness statement has now been repaired to a quotient by balanced edge scalars, but its proof still requires the same blocking machinery together with the corresponding tensor-factor uniqueness statement at a single vertex.
